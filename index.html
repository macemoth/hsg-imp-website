OICU is a context-based identification system

OICU provides explainable and robust object identification for videos, keeps track of them and stores the scene events in a temporal graph.

Situating OICU

OICU features a combination of subsymbolic and symbolic AI, harnessing the real-world performance of subsymbolic AI and the transparency of symbolic reasoning. Instead of merely classifying objects into generic classes, it creates locally unique identifiers (LUI) for every recognised object from their context. LUIs are transparent and can even be integrated in other systems.

Architecture

Our Casimir architecture consists of two large subsymbolic models on each end. They are connected by a heuristics-driven, symbolic link providing explainability, transparence, interoperability with other systems.

Scene graph generation

On its lowest level, OICU generates scene graphs using a customised transformer model from image frames. Scene graphs are transparent and machine-readable descriptions of a scene.

Symbolic link

Scene graphs from each frame are synthesised into a temporal graph, tracking identified objects as they change position and context over time. Location, their context, relations and class label are used for reidentification, achieving high accuracies.

Language model inference

Results from the sybolic link are inserted into a general-purpose language model, providing an efficient and convenient interface for users as a QA system. Want to know what happened to the laptop on the table? Just ask the model!

Frequently asked questions

What exactly makes OICU explainable?

The symbolic link connecting the scene graph generator and the language model is essentially a graph with text vertices and nodes that be visually inspected. The heuristic rules matching an object in time t-1 to one in t are human-crafted and can be changed upon desire, as any rules for triggering events can.

Aren't there any deep learning models that perform the same task end-to-end?

Yes, there are! But their end-to-end nature makes them black boxes, giving neither the user nor the developer insight into the inference process. Those models are further limited by biases and if their behaviour is to be altered, it must be implemented into the labelling and the models need to be retrained.

Can OICU do real-time processing?

Not yet. Although the symbolic link and language model are reasonably fast, the scene graph generator is a research-grade implementation and not yet optimised for real-time processing. However, translating its code into TensorFlow and using dedicated hardware should be sufficient for real-time processing.
